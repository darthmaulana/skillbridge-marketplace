Build a mobile-first web-based Android app / PWA based on the Figma design.

Project type:

* Web-based Android app / PWA
* Mobile-first responsive design
* Use the Figma design as the main UI reference
* Match spacing, layout, colors, typography, cards, buttons, badges, and navigation from the Figma design as closely as possible

Recommended tech stack:

* Next.js
* React
* TypeScript
* Tailwind CSS
* Supabase
* Supabase Auth
* Supabase PostgreSQL
* Supabase Storage
* Supabase Realtime for chat
* PWA support

App concept:
Create a job and skill marketplace where users can:

1. Post jobs when they need work done
2. Post skills when they want to offer services
3. Browse job posts and skill posts
4. Search and filter posts
5. Chat with other users
6. Verify their account for offline jobs using KTP verification
7. Report/block suspicious users
8. Admin can manually approve KTP verification

Rules:

* Online job posts can be created by normal registered users
* Offline job posts can only be created by users with approved KTP verification
* Users can only contact/apply to offline jobs if their verification is approved
* Public users only see “Verified for Offline Jobs” badge
* Never show KTP, NIK, selfie, or private identity data publicly

Security and privacy:

* Store KTP and selfie images in private Supabase Storage bucket
* Do not use public URLs for KTP/selfie
* Use signed temporary URLs for admin viewing only
* Only admin can view verification files
* Store NIK as a hash if NIK is collected
* Do not display KTP, selfie, NIK, full address, or private identity data publicly
* Add admin audit logs for verification actions
* Add report/block feature
* Add Row Level Security policies in Supabase
* Users can only edit/delete their own posts
* Admin can manage all posts/users/reports
* Delete raw verification files after the review/retention period if implemented
